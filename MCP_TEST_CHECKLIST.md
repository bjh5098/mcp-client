# MCP 연동 테스트 체크리스트

## 기본 기능 테스트

### 1. 서버 등록/조회
- [ ] 서버 추가 버튼 클릭
- [ ] 서버 이름 입력
- [ ] Transport 타입 선택 (stdio/streamable-http/sse)
- [ ] Transport별 설정 입력
  - [ ] STDIO: command, args, env
  - [ ] Streamable HTTP: url, headers
  - [ ] SSE: url, headers
- [ ] 저장 후 서버 목록에 표시되는지 확인
- [ ] localStorage에 저장되는지 확인 (개발자 도구 > Application > Local Storage)

### 2. 서버 연결
- [ ] 연결 버튼 클릭
- [ ] 연결 성공 시 상태 표시 (녹색 점)
- [ ] 연결 실패 시 에러 메시지 표시
- [ ] 연결 상태가 MCPContext를 통해 전역으로 공유되는지 확인

### 3. 서버 기능 조회
- [ ] 연결된 서버 확장 (화살표 클릭)
- [ ] Tools 목록 로드 확인
- [ ] Prompts 목록 로드 확인
- [ ] Resources 목록 로드 확인
- [ ] 기능이 없을 때 적절한 메시지 표시

### 4. Tool 실행
- [ ] Tool 카드에서 Arguments 입력
- [ ] 실행 버튼 클릭
- [ ] 결과 표시 확인
- [ ] 에러 발생 시 에러 메시지 표시

### 5. Prompt 가져오기
- [ ] Prompt 카드에서 Arguments 입력
- [ ] 가져오기 버튼 클릭
- [ ] 결과 표시 확인

### 6. Resource 읽기
- [ ] Resource 카드에서 읽기 버튼 클릭
- [ ] 내용 표시 확인

### 7. 서버 연결 해제
- [ ] 연결 해제 버튼 클릭
- [ ] 상태가 연결 해제로 변경되는지 확인
- [ ] 기능 목록이 초기화되는지 확인

### 8. 서버 수정/삭제
- [ ] 수정 버튼 클릭하여 서버 정보 변경
- [ ] 삭제 버튼 클릭하여 서버 제거
- [ ] localStorage에서도 제거되는지 확인

### 9. 설정 가져오기/내보내기
- [ ] 내보내기 버튼 클릭하여 JSON 파일 다운로드
- [ ] 가져오기 버튼 클릭하여 JSON 파일 업로드
- [ ] 서버 목록이 올바르게 복원되는지 확인

## 통합 테스트

### 10. 채팅에서 MCP 사용
- [ ] MCP 서버 연결
- [ ] MCP 도구 설정에서 서버 활성화
- [ ] 채팅에서 MCP 도구가 사용되는지 확인

### 11. 상태 동기화
- [ ] 여러 탭에서 동일한 서버 연결 상태 확인
- [ ] localStorage 변경 시 다른 탭에서 반영되는지 확인

## 문제 해결

### 일반적인 문제
1. **연결 실패**
   - 서버 설정 확인 (command, url 등)
   - 네트워크 연결 확인
   - 서버 로그 확인

2. **기능 목록이 로드되지 않음**
   - 서버가 연결되어 있는지 확인
   - 브라우저 콘솔에서 에러 확인
   - API 응답 형식 확인

3. **상태가 업데이트되지 않음**
   - MCPContext의 refreshStatuses 호출 확인
   - 주기적 갱신 (5초) 동작 확인

## 테스트 시나리오

### 시나리오 1: STDIO 서버 연결
```
1. 서버 추가
   - 이름: "Test STDIO Server"
   - Transport: stdio
   - Command: "node"
   - Args: "server.js"
2. 저장
3. 연결
4. 기능 조회
5. Tool 실행
```

### 시나리오 2: HTTP 서버 연결
```
1. 서버 추가
   - 이름: "Test HTTP Server"
   - Transport: streamable-http
   - URL: "http://localhost:3000/mcp"
2. 저장
3. 연결
4. 기능 조회
```

### 시나리오 3: 설정 백업/복원
```
1. 여러 서버 등록
2. 내보내기
3. 모든 서버 삭제
4. 가져오기
5. 서버 목록 복원 확인
```
